# Privacy and Compliance — FinTrack Pro

> **Fase:** 1 — Discovery | **Iteracao:** 1 | **Bloco:** 1.6 | **Status:** Rascunho

---

## Classificacao de Dados

| Dado | Categoria LGPD | Sensibilidade |
|------|---------------|---------------|
| Nome do titular | Pessoal | Media |
| CPF / CNPJ | Pessoal | Alta |
| E-mail e telefone | Pessoal | Media |
| Dados bancarios (agencia, conta, saldo) | Pessoal sensivel (financeiro) | Critica |
| Transacoes bancarias | Pessoal sensivel (financeiro) | Critica |

O sistema trata **dados pessoais sensiveis de natureza financeira** em larga escala. Essa classificacao determina controles de seguranca reforçados e obrigacoes especificas sob a LGPD (Lei 13.709/2018).

---

## Base Legal para Tratamento

| Aspecto | Detalhe |
|---------|---------|
| Base legal primaria | **Consentimento explicito do titular** (Art. 7, I e Art. 11, I da LGPD) |
| Mecanismo de consentimento | Fluxo gerenciado pelo banco emissor via Open Finance Brasil |
| Granularidade | Por instituicao financeira e escopo de dados (saldo, transacoes, dados cadastrais) |
| Revogacao | Titular pode revogar a qualquer momento via banco ou via FinTrack Pro |
| Registro | Todos os consentimentos sao registrados com timestamp, escopo e instituicao |

O fluxo de consentimento do Open Finance segue a regulamentacao do Banco Central, garantindo que o titular autorize explicitamente o compartilhamento antes de qualquer dado ser transmitido ao FinTrack Pro.

## Politica de Retencao

| Fase | Duracao | Acao |
|------|---------|------|
| Assinatura ativa | Enquanto o contrato estiver vigente | Dados armazenados com criptografia e acesso controlado |
| Pos-cancelamento | 90 dias | Dados mantidos para possivel recuperacao da conta |
| Apos periodo de graca | Imediato | Anonimizacao completa e irreversivel dos dados pessoais |
| Dados anonimizados | Indefinido | Utilizaveis para analises estatisticas e melhoria do produto |

---

## DPO — Encarregado de Dados

| Aspecto | Situacao |
|---------|----------|
| DPO nomeado | **Nao** |
| Previsao de nomeacao | Q3 2026 |
| Risco | **ALTO** — a LGPD exige DPO para empresas que tratam dados pessoais sensiveis em larga escala |

> **RISCO IDENTIFICADO:** A ausencia de DPO nomeado representa uma nao-conformidade com a LGPD. Recomendacao forte: nomear DPO (interno ou terceirizado) **antes do lancamento do MVP**. Este item deve ser escalado ao CEO como bloqueante de compliance.

---

## Criptografia

| Contexto | Algoritmo | Detalhe |
|----------|-----------|---------|
| Dados em repouso | AES-256 | Criptografia no PostgreSQL (RDS) com chaves gerenciadas via AWS KMS |
| Dados em transito | TLS 1.3 | Todas as comunicacoes externas e internas entre servicos |
| Backups | AES-256 | Backups do RDS criptografados com mesma chave KMS |
| Chaves de API | Vault / Secrets Manager | Credenciais de Open Finance e LLM nunca armazenadas em codigo |
