# Report Region Rules

Regras de cada region de relatório, organizadas por seção. Cada subpasta contém arquivos `.md` que definem ID, schema, campos obrigatórios e template visual de cada region.

## Contents

- `backlog/` — Épicos, stories, dependências, go/no-go
- `domain/` — Regions específicas por context-template (AI/ML, data, mobile, etc.)
- `executive/` — Overview one-pager, premises, next steps, go/no-go decision
- `financial/` — TCO, break-even, effort estimation, cost per component
- `metrics/` — KPIs de negócio e técnicos, SLAs, DORA, targets por fase
- `narrative/` — História do projeto, condições, assinaturas
- `organization/` — Equipe, stakeholders, RACI, metodologia, on-call
- `privacy/` — LGPD/GDPR, dados pessoais, DPO, retenção, sub-processors
- `product/` — Problema, personas, jornadas, valor, roadmap, escopo
- `quality/` — Auditor score, 10th-man, gaps, checklist
- `research/` — Entrevistas, quotes, oportunidades, dados quantitativos
- `risk/` — Matriz de riscos, riscos técnicos, hipóteses, feasibility
- `security/` — Autenticação, criptografia, compliance, data classification
- `technical/` — Stack, arquitetura macro, ADRs, integrações, containers, NFRs
