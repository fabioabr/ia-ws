# TCO and Build vs Buy — FinTrack Pro

> **Fase:** 1 — Discovery | **Iteracao:** 1 | **Bloco:** 1.8 | **Status:** Rascunho

---

## Alternativas Analisadas

### Alternativa 1 — Comprar (Conta Azul Enterprise)

| Aspecto | Avaliacao |
|---------|-----------|
| Solucao | Conta Azul Enterprise + customizacao |
| Custo estimado | R$1.200/mes + custo de customizacao |
| Vantagem | Time-to-market rapido, suporte incluso |
| Limitacao critica | Nao suporta Open Finance nativo |
| Limitacao critica | Sem projecoes com IA |
| Limitacao critica | Categorizacao limitada para cenarios multi-banco complexos |
| Veredicto | **Descartada** — nao atende aos diferenciais competitivos do produto |

### Alternativa 2 — Adaptar (Metabase + API Open Finance)

| Aspecto | Avaliacao |
|---------|-----------|
| Solucao | Metabase (open source) como dashboard + integracao direta com API de Open Finance |
| Custo estimado | Baixo (infra + desenvolvimento de conectores) |
| Vantagem | Custo reduzido, dashboards flexiveis |
| Limitacao critica | Categorizacao manual permanece — nao resolve o problema central |
| Limitacao critica | Sem camada de IA para projecoes e anomalias |
| Limitacao critica | Experiencia de usuario generica, nao otimizada para fluxo financeiro |
| Veredicto | **Descartada** — resolve parcialmente o problema mas nao entrega o diferencial |

### Alternativa 3 — Construir (FinTrack Pro Custom)

| Aspecto | Avaliacao |
|---------|-----------|
| Solucao | Desenvolvimento proprio com stack Node.js TS + React + PostgreSQL |
| Custo estimado | R$3.86M em 3 anos (ver TCO abaixo) |
| Vantagem | Controle total sobre diferenciais competitivos (IA + Open Finance nativo) |
| Vantagem | Propriedade intelectual completa |
| Vantagem | Flexibilidade para evoluir conforme demanda do mercado |
| Risco | Maior investimento inicial e time-to-market mais longo |
| Veredicto | **Selecionada** — unica opcao que entrega o diferencial competitivo completo |

---

## Decisao

**Build (construir).** O diferencial competitivo do FinTrack Pro — consolidacao multi-banco via Open Finance + projecoes com IA + alertas inteligentes — nao existe como produto pronto no mercado brasileiro. Depender de terceiros para o core do produto eliminaria a vantagem competitiva e limitaria a capacidade de iteracao.

O cliente (CEO) validou: *"O diferencial de projecoes com IA e o core do produto. Nao podemos depender de terceiros para isso."*

## Composicao de Custos

| Categoria | % do Total | Observacao |
|-----------|-----------|------------|
| Equipe | 77,2% | Maior custo; 5 pessoas com reajuste de 10% a.a. |
| Contingencia | 13,0% | Buffer de 15% sobre custos operacionais |
| AWS | 4,7% | Crescimento proporcional a base de clientes |
| Open Finance API | 2,8% | Custo por conexao bancaria ativa |
| LLM API | 1,9% | Custo por chamada; escala com uso de projecoes |
| Licencas | 0,5% | Ferramentas de desenvolvimento, monitoramento |

A equipe representa mais de 3/4 do custo total, o que e tipico para produtos SaaS em estagio inicial. A otimizacao de custos no medio prazo depende de automacao e ganho de escala, nao de reducao de equipe.

## Calculo de Break-Even

| Variavel | Valor |
|----------|-------|
| Custo mensal medio (Ano 1) | R$93.725 (R$1.124.700 / 12) |
| Ticket medio considerado | R$499/mes (plano Pro) |
| Margem bruta estimada | ~85% (SaaS com custos variaveis baixos) |
| Receita liquida por cliente | R$424/mes |
| **Clientes necessarios para break-even** | **~221 clientes Pro** |
| Meta OKR em 6 meses | 100 empresas pagantes |
| Break-even estimado | Entre o mes 12 e o mes 18 pos-lancamento |

O calculo considera receita exclusivamente do plano Pro. Na pratica, a distribuicao entre Starter (R$199), Pro (R$499) e Enterprise (R$999) pode acelerar ou retardar o break-even dependendo do mix de clientes.

## Premissas e Riscos

| # | Premissa | Risco se invalida |
|---|----------|-------------------|
| 1 | Reajuste salarial de 10% a.a. para a equipe | Custos de equipe podem ser maiores em mercado aquecido de tech |
| 2 | Crescimento de uso proporcional ao numero de clientes | Picos de uso podem gerar custos de infra acima do previsto |
| 3 | Creditos AWS Activate cobrindo Ano 1 | Se creditos acabarem antes, custo de infra do Ano 1 aumenta |
| 4 | Custo de LLM API estavel ou decrescente | Mudancas de pricing de OpenAI/Anthropic impactam diretamente |
| 5 | Equipe de 5 pessoas suficiente para o MVP | Atraso no MVP se houver turnover ou necessidade de contratacao |
| 6 | 100 clientes pagantes em 6 meses (OKR) | Break-even se estende se a adocao for mais lenta |
| 7 | Open Finance API com disponibilidade > 99.5% | Indisponibilidade do provider impacta a proposta de valor central |

> **Nota:** O TCO nao inclui custos de marketing, vendas e suporte ao cliente. Esses custos devem ser adicionados em uma analise financeira completa do negocio.
