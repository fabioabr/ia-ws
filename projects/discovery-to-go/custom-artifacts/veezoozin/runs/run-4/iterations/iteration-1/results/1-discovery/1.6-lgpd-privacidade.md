> O Veezoozin não persiste os dados retornados pelo BigQuery (são transientes — exibidos na UI e descartados). Este é o maior risco de privacidade do projeto.

### 6.5 Fronteiras de Privacidade

#### Permitido

- Criptografia AES-256 at-rest para todos os dados pessoais (default GCP)
- Armazenamento de dados em região Brasil (southamerica-east1)
- Logs de acesso com retenção de 12 meses
- Cache transiente de resultados de queries (não persistir PII)
- Envio de dados para APIs de LLM externas COM consentimento informado do tenant
- Uso de histórico de queries para melhoria do serviço COM base em legítimo interesse

#### Proibido

- Persistência de dados retornados pelo BigQuery do cliente além do cache transiente
- Envio de colunas marcadas como PII pelo admin para LLM externo SEM opt-in explícito
- Compartilhamento de dados de um tenant com outro (cross-tenant leak)
- Armazenamento de API keys de LLM em texto plano (obrigatório Secret Manager)
- Retenção indefinida de histórico de queries (máximo 12 meses, depois anonimizar ou excluir)
- Uso de dados de queries para treinamento/fine-tuning de modelos (mesmo anonimizados, sem consentimento)
- Transferência de dados pessoais para fora do Brasil sem cláusulas contratuais adequadas

#### Observações

- Sub-operadores identificados: Anthropic (Claude API), Google (Gemini API, Vertex AI), OpenAI, Stripe
- DPAs necessários com TODOS os sub-operadores antes do lançamento
- Google Cloud já possui DPA padrão para GCP; verificar para Vertex AI especificamente
- Anthropic e OpenAI possuem DPAs disponíveis (verificar termos de data retention)
- Stripe possui DPA embutido nos termos de serviço

## 2. Análise do Especialista

### Classificação de risco de privacidade: MÉDIO-ALTO

O Veezoozin tem risco de privacidade MÉDIO-ALTO pelos seguintes fatores:

1. **Dados de terceiros em trânsito para LLMs externos** — O maior risco. Dados do BigQuery do cliente (que podem conter PII dos clientes do cliente) passam pelo Veezoozin e são enviados para APIs externas. O Veezoozin é simultaneamente controlador (dos dados de seus usuários) e operador (dos dados dos clientes dos tenants).

2. **Armazenamento de credenciais sensíveis (BYOK)** — API keys de LLM e credenciais BigQuery são alvos de alto valor. Leak = custo financeiro + perda de confiança.

3. **Multi-tenant com row-level isolation** — Row-level é o modelo mais eficiente mas também o mais arriscado: um bug no middleware de tenant_id = cross-tenant data leak.

4. **Equipe de 1 pessoa** — Sem DPO, sem comitê de privacidade, sem second pair of eyes para review de segurança. Risco operacional alto.

### Riscos de Privacidade Top 5

| # | Risco | Probabilidade | Impacto | Tag |
|---|-------|-------------|---------|-----|
| 1 | **Cross-tenant data leak** — Bug no middleware expõe dados do Tenant A para Tenant B | Média (20%) | Crítico — perda de confiança, LGPD violation | [INFERENCE] |
| 2 | **PII enviada para LLM externo sem consentimento** — Dados do BigQuery com PII são enviados para Claude/Gemini sem opt-in | Alta (40%) | Alto — violação de base legal LGPD | [INFERENCE] |
| 3 | **API key leakage (BYOK)** — Credenciais de LLM do tenant expostas via log ou error message | Baixa (10%) | Crítico — custo financeiro para o tenant | [INFERENCE] |

| Campo | Detalhe |

| **Responsável** | Fabio |

| **Prazo** | Sprint 1-2 (fundação — deve ser a primeira coisa implementada) |
| **Consequência se não resolver** | Vazamento de dados entre tenants = violação LGPD + perda de clientes + possível multa de até 2% do faturamento |

## 3. Recomendações de Segurança

### R1 — Implementar pseudonimização de PII antes do envio ao LLM

**Proposta:** Sistema de masking que detecta e substitui PII antes de enviar contexto para APIs de LLM. Admin marca colunas como PII no schema mapping. Sistema substitui valores por placeholders durante o processamento.

**Classificação:** Mandatório para MVP
**Justificativa:** Sem isso, PII de clientes dos tenants é enviada para APIs externas sem controle. Violação direta da LGPD.

### R2 — Criar Política de Privacidade e Termos de Uso antes do lançamento

**Proposta:** Redigir com apoio jurídico: (1) Política de Privacidade com listagem de dados tratados, bases legais, terceiros, direitos do titular; (2) Termos de Uso com cláusula de BYOK, responsabilidade sobre dados, SLA; (3) Política de Cookies.

**Classificação:** Mandatório — blocker legal
**Justificativa:** SaaS que trata dados pessoais sem política de privacidade está em violação da LGPD desde o primeiro cadastro.

**Ação:** Contratar consultoria jurídica para redigir documentos
**Responsável:** Fabio
**Prazo:** Semana 12-14 (antes do lançamento)
**Custo:** R$3-8K (consultoria jurídica para SaaS)
**Consequência se não resolver:** Lançar sem política de privacidade = infração LGPD desde o dia 1

### R3 — Verificar obrigatoriedade de DPO e enquadramento como agente de pequeno porte

**Proposta:** Consultar jurista especializado em LGPD para verificar se mAInd Tech (startup de 1 pessoa) se enquadra na Resolução CD/ANPD n.2/2022 (agentes de tratamento de pequeno porte). Se sim, documentar a dispensa. Se não, contratar DPO terceirizado.

**Classificação:** Mandatório — verificação legal
**Justificativa:** Art. 41 LGPD exige DPO. Operar sem saber se precisa é risco desnecessário.

### R4 — Testes automatizados de isolamento multi-tenant

**Proposta:** Suite de testes de integração que, para cada endpoint da API, verifica que Tenant A autenticado NUNCA recebe dados de Tenant B. Rodar em CI/CD a cada deploy.

**Classificação:** Mandatório para MVP
**Justificativa:** Cross-tenant leak é o risco #1 de privacidade. Teste automatizado é a única forma de prevenir regressão.

### R5 — DPA com todos os sub-operadores antes do lançamento

**Proposta:** Assinar DPA (Data Processing Agreement) com: Anthropic, Google (para Gemini e GCP), OpenAI, Stripe. Verificar cláusulas de retenção de dados e residência.

**Classificação:** Mandatório — blocker legal
**Justificativa:** LGPD exige contrato formal com operadores que processam dados pessoais em nome do controlador.

**Ação:** Verificar DPAs disponíveis e assinar
**Responsável:** Fabio
**Prazo:** Semana 10-12
**Custo:** Zero (DPAs padrão são gratuitos) ou R$1-3K se precisar de análise jurídica
**Consequência se não resolver:** Dados pessoais sendo processados por terceiros sem base contratual = violação LGPD
