# Bloco #6 — Privacidade e Compliance

> Resultado da entrevista simulada (Fase 1 — Discovery). **Modo profundo (deep) ativado** — o Veezoozin processa dados de negócio de terceiros (tenants) que podem conter PII. O modelo BYOK adiciona complexidade: as API keys dos tenants transitam pelo sistema, e os dados dos tenants são enviados a LLMs de terceiros. Utiliza como input todos os blocos anteriores (1-5). **Execução paralela com Bloco #5 (Tecnologia e Segurança).**

### 1.4. BYOK — Implicações de Privacidade

> **Referência Bloco #1 — D1.3:** BYOK obrigatório a partir do Pro.
> **Referência Bloco #5:** API keys no Secret Manager, criptografia AES-256.

O modelo BYOK cria um cenário de privacidade **único**: os dados do tenant são enviados para um LLM externo usando a chave **do próprio tenant**. Isso altera a dinâmica de responsabilidade.

| Aspecto | Implicação | Mitigação | Fonte |
|---------|-----------|-----------|-------|
| **Dados no prompt** | O prompt enviado ao LLM contém: schema do banco, glossário, pergunta NL. Pode conter PII se o schema tiver colunas com nomes pessoais. | Minimizar PII no prompt. Enviar schema abstrato (column types, não sample data). | [INFERENCE] |
| **Resultado no prompt** | Na etapa 6 (insight generation), o resultado da query (que pode conter PII) é enviado ao LLM. | Limitar volume de dados enviados (top 100 rows). Anonimizar se possível. | [INFERENCE] |
| **Data residency** | Dados são enviados para APIs dos providers (Anthropic nos EUA, Google nos EUA/EU, OpenAI nos EUA). | Informar no DPA. Para tenants com restrição de residência, recomendar Gemini (opção EU). | [INFERENCE] |
| **Retenção pelo provider** | Anthropic e OpenAI: dados de API não são usados para treino (opt-out padrão para API). Google: Data Processing Addendum. | Documentar DPAs dos providers. Exigir que tenant revise termos do seu provider. | [INFERENCE] |
| **Responsabilidade** | Com BYOK, o tenant é quem contrata o LLM (é a key dele). Veezoozin é intermediário técnico. | DPA claro: Veezoozin processa dados sob instrução do tenant. Tenant é responsável pelo contrato com o provider de LLM. | [INFERENCE] |
| **Pool Free (Gemini Flash)** | No tier Free, a mAInd Tech é quem contrata o Gemini. Dados dos tenants Free são enviados pela key da mAInd Tech. | DPA com Google. Termos de uso do Free devem informar que dados são processados via Gemini Flash da mAInd Tech. | [INFERENCE] |

## Parte 2 — Análise do Especialista

### LGPD e o Modelo BYOK — Uma Dinâmica Incomum

O modelo BYOK cria uma situação juridicamente **interessante e pouco usual**:

1. **Com BYOK (Pro/Enterprise):** O tenant é quem contrata o LLM (é a key dele). O Veezoozin é apenas o intermediário técnico que executa a chamada. A responsabilidade pelo processamento dos dados no LLM é **compartilhada**: o Veezoozin decide o formato do prompt (operador), mas o tenant decidiu usar aquela API key naquela LLM (controlador). Isso precisa estar claro no DPA.

2. **Sem BYOK (Free/Trial):** A mAInd Tech é quem contrata o Gemini Flash. Nesse caso, a mAInd Tech é plenamente responsável pelo processamento de dados dos tenants Free no LLM. Isso é um risco maior, mitigado pelo limite de 50 queries/mês e pela natureza limitada do plano Free (1 dataset, 5 tabelas).

3. **Ponto crucial:** Os dados do tenant (schema, glossário, resultados de queries) **transitam pelo sistema Veezoozin** independentemente do BYOK. Mesmo com a key do tenant, é o Cloud Run do Veezoozin que faz a chamada. Isso torna o Veezoozin **operador** para fins de LGPD.

### Pragmatismo para Startup de 1 Pessoa

A LGPD exige conformidade, mas **não exige perfeição no dia 1**. Para uma startup pré-receita, a estratégia pragmática é:

1. **Antes do launch (obrigatório):**
   - Política de Privacidade publicada no site
   - Termos de Uso com cláusulas de dados
   - Email de DPO (privacidade@veezoozin.com)
   - Registro simplificado de operações de dados

2. **Com os primeiros clientes Pro (necessário):**
   - DPA template para assinatura dos tenants
   - Funcionalidade de export/delete de dados
   - Purge automático por tier funcionando

3. **Com clientes Enterprise (crítico):**
   - DPA customizável
   - Relatório de impacto à proteção de dados (RIPD)
   - Auditoria de segurança sob demanda
   - DPO terceirizado (se MRR >R$ 25K)

### Decisão Automatizada (Art. 20 LGPD)

O Veezoozin **não toma decisões automatizadas sobre titulares de dados**. Ele gera consultas SQL e visualizações. A decisão é tomada pelo humano (Marina, Rafael) que interpreta o resultado.

No entanto, há um cenário de risco: se um tenant usar o Veezoozin para queries como "quais funcionários devem ser demitidos baseado em performance?" — o resultado poderia ser usado para uma decisão automatizada. Isso é responsabilidade do tenant (controlador), mas o Veezoozin deve:
1. Incluir aviso nos Termos de Uso
2. Não se posicionar como ferramenta de decisão sobre pessoas
3. Nos insights gerados, nunca recomendar ações sobre pessoas específicas

## Parte 3 — Recomendações

### R1. Minimizar PII nos prompts enviados ao LLM

**Problema:** O prompt do pipeline NL-to-SQL inclui schema, glossário e (na etapa 6) resultados de queries. Se o tenant tem tabelas com dados pessoais (clientes, funcionários, salários), esses dados podem ser enviados ao LLM.

**Recomendação em 3 níveis:**

| Nível | Ação | Quando implementar |
|:-----:|------|:------------------:|
| **1. Schema abstrato** | No prompt, enviar apenas nome da coluna e tipo (não sample data). Ex: `clientes(id INT, nome VARCHAR, cpf VARCHAR)` — sem valores reais. | MVP |
| **2. Result anonymization** | Na etapa 6 (insight), anonimizar PII antes de enviar ao LLM: substituir nomes por "Pessoa 1", CPFs por "XXX". | Fase 2 |
| **3. Column tagging** | Permitir que o Admin do Tenant marque colunas como "PII". Colunas marcadas são excluídas do prompt ou anonimizadas automaticamente. | Fase 2 |

### R2. Criar DPA template antes do launch

**Problema:** Sem DPA, o Veezoozin processa dados de terceiros sem acordo formal. Risco legal e comercial (Carla — Gatekeeper — exige DPA para aprovar).

**Recomendação:**
- Criar DPA template baseado em modelos open-source (ex: GDPR.eu, Associação Brasileira de Startups)
- Incluir: finalidade, dados processados, sub-processadores (GCP, Stripe, LLM providers), retenção, direitos
- Disponibilizar como parte do signup Pro/Enterprise (aceitação digital)
- Custo: R$ 0 (template) a R$ 2.000-5.000 (revisão por advogado — recomendado antes do 1o Enterprise)

### R3. Implementar "Right to be Forgotten" automatizado

**Problema:** LGPD Art. 18, VI — titular pode solicitar eliminação de dados. Com 1 dev, isso precisa ser automatizado.

**Recomendação:**
- **Botão "Deletar minha conta"** no painel do usuário
- **Cascade delete:** conta → queries do Firestore → embeddings do Vertex AI → sessions → logs (marca como "anonimizado")
- **Preservar:** logs de auditoria (anonimizados — remove PII, mantém ação), dados fiscais no Stripe (obrigação legal)
- **Prazo:** processamento em até 15 dias (LGPD), mas implementar para ser imediato
- **Confirmação:** email de confirmação da eliminação

### R4. Termos de Uso com cláusula explícita sobre LLM e BYOK

**Problema:** O usuário precisa saber que seus dados são processados por um LLM externo. Com BYOK, a dinâmica de responsabilidade muda.

**Recomendação — Cláusulas obrigatórias nos Termos de Uso:**

1. "Seus dados são processados por provedores de inteligência artificial (LLMs) para geração de consultas SQL e insights."
2. "No plano Pro/Enterprise, você fornece sua própria API key. O Veezoozin utiliza essa key para processar suas consultas. Você é responsável pelos termos de uso do seu provedor de LLM."
3. "No plano Free, seus dados são processados pelo Google Gemini Flash, contratado pela mAInd Tech. Os dados não são usados para treinamento de modelos."
4. "O Veezoozin não armazena os dados do seu banco de dados. As consultas são executadas em tempo real e os resultados não são persistidos além do cache temporário."

### R5. Caminho pragmático de compliance LGPD para 1 pessoa

**Recomendação — Roadmap de compliance por estágio:**

| Estágio | MRR | Ações LGPD | Custo estimado |
|---------|:----|-----------|---------------:|
| **Pré-launch** | R$ 0 | Política de Privacidade, Termos de Uso, email DPO, registro simplificado de dados | R$ 0 (templates) |
| **Primeiros clientes** | <R$ 5K | DPA template, botão "deletar conta", purge automático, aviso de LLM nos termos | R$ 0-2K (revisão jurídica opcional) |
| **Tração** | R$ 5-17K | DPA customizável, column tagging de PII, auditoria simplificada | R$ 2-5K (advogado para revisão) |
| **Break-even** | R$ 17-30K | DPO terceirizado (~R$ 2K/mês), RIPD formal, teste de incidente | R$ 2K/mês |
| **Escala** | R$ 30K+ | DPO dedicado ou consultoria permanente, certificação ISO 27001 (se Enterprise exigir) | R$ 5-10K/mês |

**Princípio:** Compliance cresce com a receita. Não gastar R$ 10K em advogado antes de ter R$ 1 de receita. Mas nunca lançar sem o mínimo (Política de Privacidade + Termos + email DPO).
