## Parte 2 — Análise do Especialista

### Viabilidade Financeira do Modelo

O modelo econômico do Veezoozin no run-3 é **estruturalmente superior** ao de um SaaS convencional:

1. **Custo fixo radicalmente baixo (R$ 17K/mês).** Um SaaS B2B típico com time de 5-8 pessoas tem custo fixo de R$ 60-120K/mês. O Veezoozin precisa de 5-7x menos receita para atingir break-even.

2. **BYOK elimina o maior risco de margem.** Em SaaS com IA, o custo de LLM pode representar 30-60% da receita. Com BYOK, esse custo é zero para a mAInd Tech. A margem bruta por tenant fica em 80-92%.

3. **Pay-per-use elimina o "death valley".** Infraestrutura serverless significa custo quase zero sem clientes. Não há queima de caixa em infra antes de ter receita.

4. **Break-even acessível (~27 Pro + 3 Enterprise).** Em termos de mercado, 30 clientes pagantes é atingível em 14-18 meses para um SaaS B2B com produto diferenciado.

### Riscos da Projeção

| Risco | Impacto na projeção | Mitigação |
|-------|---------------------|-----------|
| Conversão Free-Pro abaixo de 5% | Break-even atrasa 6+ meses | Limitar agressivamente o Free, demonstrar valor claro no Pro |
| Churn mensal acima de 8% | MRR não cresce, estabiliza ou cai | Feedback loop, sucesso do cliente, "guided first query" |
| Ticket médio Enterprise abaixo de R$ 2.497 | Precisa de mais Pro para compensar | Manter valor mínimo, não dar desconto no Enterprise |
| Custo do pool Free escala sem conversão | CAC sobe sem retorno | Limitar pool Free agressivamente (30 queries/mês no lugar de 50) |

### Análise do Pricing

O preço de R$ 497/mês para o Pro está **bem posicionado**:

- **Abaixo de ThoughtSpot** (US$ 1.250+/mês) — acessível para PMEs
- **Acima de Metabase Cloud** (US$ 85/mês) — justificável pelo NL + contexto + multi-idioma
- **Na faixa de Lightdash/Preset** (US$ 300-500/mês) — competitivo por feature

**Preocupação:** O plano Enterprise a R$ 2.497 pode ser baixo para empresas que pagam R$ 6.000+/mês em ThoughtSpot. Considerar precificação variável por número de usuários ou volume de queries para capturar mais valor de contas grandes.

## Parte 3 — Recomendações

### R1. Implementar precificação Enterprise variável

**Problema:** Enterprise fixo em R$ 2.497 subvaloriza contas grandes. Uma empresa com 500 usuários paga o mesmo que uma com 20.

**Recomendação:** Modelo Enterprise com base + variável:
- **Base:** R$ 2.497/mês (até 50 usuários)
- **Usuário adicional:** R$ 29/mês
- **Exemplo:** Enterprise com 200 usuários = R$ 2.497 + (150 x R$ 29) = R$ 6.847/mês

Isso captura mais valor de contas grandes sem perder PMEs que precisam de features Enterprise.

### R2. Oferecer trial de 14 dias no Pro (sem exigir API key)

**Problema:** Pedir BYOK desde o dia 1 do Pro cria fricção. O decisor quer testar antes de comprometer infra.

**Recomendação:**
- **Trial Pro de 14 dias** com pool compartilhado (200 queries)
- Após 14 dias: converte para Pro pago com BYOK obrigatório, ou downgrada para Free
- Custo do trial: ~R$ 2-6 por tenant (200 queries x R$ 0,01-0,03)
- Se 50 tenants fazem trial/mês: R$ 100-300/mês de CAC

**Impacto esperado:** Conversão Free-Pro aumenta de 10% para 15-20%.

### R3. Criar "Usage Alerts" antes de limites

**Problema:** Se o tenant Free atinge 50 queries sem aviso, a experiência é "parede". Se o Pro tem query pesada que consome muito do BigQuery, o custo surpreende.

**Recomendação:**
- **Free:** Alerta em 35 queries (70%) e 45 queries (90%). Upsell message: "Quer queries ilimitadas? Conheça o Pro."
- **Pro:** Alerta de consumo BigQuery quando custo estimado >R$ 100/mês. Dashboard de consumo mensal.
- **Enterprise:** Alertas customizáveis por admin.

### R4. Definir política de dunning clara

**Problema:** Pagamento falhado sem tratamento = churn involuntário.

**Recomendação:**
1. **Dia 0:** Pagamento falha. Retry automático via Stripe.
2. **Dia 3:** 2o retry. Email para admin: "Atualize seu cartão".
3. **Dia 7:** 3o retry. Conta fica em "modo leitura" — pode consultar histórico, não pode fazer novas queries.
4. **Dia 14:** Conta suspensa. Dados preservados por 90 dias.
5. **Dia 90:** Dados deletados conforme LGPD.

### R5. Projetar métricas de LTV e CAC desde o MVP

**Recomendação:** Mesmo no MVP, instrumentar:

| Métrica | Fórmula | Meta |
|---------|---------|:----:|
| **LTV (Pro)** | R$ 497 x (1 / churn mensal) | >R$ 9.940 (20 meses) |
| **LTV (Enterprise)** | R$ 2.497 x (1 / churn mensal) | >R$ 49.940 (20 meses) |
| **CAC** | (Custo pool Free + marketing) / novos clientes Pro | <R$ 500 |
| **LTV/CAC** | LTV / CAC | >10x |
| **Payback** | CAC / (MRR - custo variável/tenant) | <2 meses |

Meta de LTV/CAC >10x é realista porque o CAC é muito baixo (produto-led growth, sem equipe de vendas).

### R6. Considerar plano anual com desconto

**Recomendação:** Oferecer desconto de 2 meses no plano anual:
- **Pro anual:** R$ 4.970/ano (R$ 414/mês efetivo) — economia de R$ 994
- **Enterprise anual:** Negociação caso a caso

**Por que importa:**
- Reduz churn (comprometimento de 12 meses)
- Melhora cash flow (receita antecipada)
- Facilita projeção de receita
