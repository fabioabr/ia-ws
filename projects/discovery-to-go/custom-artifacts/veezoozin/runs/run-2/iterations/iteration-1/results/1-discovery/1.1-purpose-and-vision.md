# Bloco #1 — Propósito e Visão

> Resultado da entrevista simulada (Fase 1 — Discovery) entre PO e Customer simulado, cobrindo propósito do produto, problema central, visão de futuro e posicionamento competitivo.

---

## 1. Problema Central

O Veezoozin resolve um problema estrutural presente em empresas de todos os portes: **dados valiosos existem, mas estão presos atrás de barreiras técnicas**.

O problema se manifesta em quatro dimensões:

| Dimensão | Descrição | Impacto |
|----------|-----------|---------|
| **Barreira técnica** | Gestores e executivos dependem de times de dados para obter respostas. Uma pergunta simples vira um ticket que leva dias. | Decisões atrasadas, custo operacional alto |
| **Falta de contexto de negócio** | Ferramentas de BI genéricas não entendem o vocabulário do cliente. "Churn" significa coisas diferentes em cada empresa. | Respostas imprecisas, retrabalho |
| **Dados sem ação** | O usuário recebe tabelas brutas sem insights. Falta a camada de interpretação: gráficos, tendências, anomalias, recomendações. | Dados acessados mas não utilizados |
| **Multi-idioma** | Empresas latinas operam em PT-BR, EN-US e Espanhol. O sistema precisa entender perguntas em qualquer idioma. | Exclusão de usuários, adoção limitada |

**Síntese:** O problema não é falta de dados — é falta de acesso democratizado, contextualizado e acionável a esses dados.

## 3. Visão do Produto

**Declaração de visão:**

> Ser a camada de inteligência entre as pessoas e seus dados — tornando qualquer colaborador capaz de consultar, entender e agir sobre os dados da empresa, em qualquer idioma, com contexto de negócio.

**Posicionamento:** "Pergunte para seus dados" — democratização do acesso a dados via linguagem natural, com contexto de negócio por tenant, multi-idioma nativo (PT-BR/EN/ES), e integração com fontes de conhecimento externas via MCP.

### Visão em camadas

```
Camada 1 (MVP)     → Pergunta NL → SQL → Visualização (BigQuery, 3 idiomas)
Camada 2 (Fase 2)  → Multi-banco (PostgreSQL, MySQL, etc.) + MCPs + RAG
Camada 3 (Fase 3)  → Análises preditivas, alertas automáticos, agentes autônomos
```

## 4. Diferenciação Competitiva

| Concorrente | Limitação principal | Vantagem do Veezoozin |
|-------------|--------------------|-----------------------|
| Tableau Ask Data | Funciona apenas dentro do ecossistema Tableau, suporte limitado a idiomas | Standalone, multi-idioma nativo (PT-BR/EN/ES) |
| ThoughtSpot | Pricing enterprise elevado, schema rígido | Acessível para PMEs, schema flexível com aprendizado por tenant |
| ChatGPT + SQL | Sem contexto de negócio, sem multi-tenancy, sem persistência | Glossário por tenant, histórico de conversas, aprendizado contínuo |
| Metabase | Abordagem dashboard-first, não conversacional | Conversational-first com output visual rico |

**Moat principal:** Contexto de negócio por tenant + multi-idioma nativo + integração com fontes de conhecimento externas via MCP. Nenhum concorrente oferece as três coisas juntas.

## 6. Modelo Comercial (Visão Inicial)

> [!note] Detalhamento completo no Bloco #3 (Pricing & Tiers). Aqui registramos apenas a diretriz estratégica extraída da entrevista.

**Diretriz:** Modelo tiered com opção freemium.

| Tier | Propósito | Características-chave |
|------|-----------|----------------------|
| **Free** | Aquisição e viralidade | Recursos muito limitados (poucas queries/dia, 1 fonte, sem MCP) |
| **Pro** | Monetização de PMEs | Queries ilimitadas, múltiplas fontes, glossário, export |
| **Enterprise** | Grandes contas, SLA diferenciado | SSO, SLA 99.9%, database dedicado, suporte prioritário |

**Monetização por consumo:** Queries, storage de contexto, número de tenants — billing híbrido (assinatura base + usage).

## 7. Roadmap em Alto Nível

| Fase | Prazo | Escopo principal |
|------|-------|-----------------|
| **MVP** | 4 meses (12 semanas) | NL → SQL → Visualização. BigQuery apenas. 3 idiomas. 5 tenants. Glossário básico. Plano Free + Pro. |
| **Fase 2** | +3 meses | Multi-banco (PostgreSQL, MySQL, SQL Server). MCPs. RAG externo. SSO. Plano Enterprise. |
| **Fase 3** | +3 meses | Análises preditivas. Alertas automáticos. Agentes autônomos. Marketplace de conectores. |

### Fora do escopo (confirmado)

- Escrita/modificação de dados nos bancos do cliente (apenas leitura)
- Treinamento/fine-tuning de LLM próprio (usar APIs: Claude, Gemini)
- ETL/ingestão de dados
- App mobile nativo (web responsivo no MVP)
- Integração direta com ERPs/CRMs (apenas via banco de dados ou MCP)

## 8. Riscos Identificados neste Bloco

| Risco | Probabilidade | Impacto | Mitigação sugerida |
|-------|:------------:|:-------:|---------------------|
| Precisão de NL-to-SQL abaixo de 85% | Média | Alto | Feedback loop agressivo, glossário bem alimentado, fallback para sugestões |
| Adoção do plano Free sem conversão | Alta | Médio | Limitações claras no Free, valor percebido forte no Pro |
| Concorrentes reduzirem gap de multi-idioma | Baixa | Médio | Investir no contexto por tenant como moat principal |
| Custo de LLM APIs por query elevado | Média | Alto | Cache inteligente, queries similares detectadas, limites por plano |
