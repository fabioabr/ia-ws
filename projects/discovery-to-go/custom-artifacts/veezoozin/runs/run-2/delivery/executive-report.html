Pipeline NL-to-SQL (8 Etapas) REG-TECH-02
#EtapaResponsabilidadeLatencia alvo
1InputDetectar idioma, normalizar texto< 100ms
2Context AssemblySchema + glossario + historico + permissoes via embeddings< 500ms
3SQL GenerationLLM gera SQL (Claude primario, Gemini fallback)< 2s
4Validate + Guardsqlglot parse, AST validation, RLS injection, read-only check< 200ms
5ExecuteBigQuery read-only via service account do tenant< 2s (p50)
6Result ProcessDetectar tipo grafico, formatar dados, paginar< 300ms
7Insight GenerationLLM interpreta resultados com privacy guard (D35)< 1,5s
8OutputJSON: dados + grafico + insight + sugestao—
Latencia total alvo: < 5 segundos (p50)
Cache de 4 camadas: exact match (15%), semantic match (25%), schema cache, result cache. Economia projetada de 30-40% em chamadas LLM.
Alerta 10th-man: Cache hit rate projetado de 35-50% pode ser otimista. Cache e por-tenant (isola volume), TTL de 24h em dados transacionais serve dados desatualizados. Projecao conservadora: 15-20%.
