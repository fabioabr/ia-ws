## 3. Build vs Buy — Analise por Componente

### 3.1 NL-to-SQL Engine

| Criterio | Build Custom | Vanna.ai | SQLCoder / DIN-SQL | Text2SQL via LLM (Claude/Gemini) |
|----------|-------------|----------|--------------------|---------------------------------|
| **Descricao** | Pipeline custom: prompt engineering + RAG + fine-tuning de prompts | Framework open-source para NL-to-SQL com RAG integrado | Modelos especializados em SQL generation | Usar LLM generalista com prompt engineering avancado |
| **Custo de implementacao** | Alto ($40K-60K, 3-4 meses) | Baixo ($5K-10K, 2-4 semanas) | Medio ($15K-25K, 1-2 meses) | Baixo-Medio ($10K-20K, 3-6 semanas) |
| **Custo operacional/mes** | $50-100 (infra) | $0 (open-source) + LLM API | $200-500 (GPU se self-hosted) | LLM API apenas ($95-4,560/mes conforme escala) |
| **Precisao (SQL correto)** | 75-85% (sem fine-tuning) | 80-90% (com training data) | 85-92% (benchmarks academicos) | 80-88% (com bom prompt + contexto) |
| **Multi-tenant** | Total controle | Requer adaptacao | Nao nativo | Total controle via prompts |
| **Contexto de negocio** | Total controle | Suporta via RAG | Limitado | Excelente via system prompts |
| **Multi-idioma** | Depende do LLM | Depende do LLM | Limitado (EN mainly) | Nativo (Claude/Gemini) |
| **Vendor lock-in** | Nenhum | Baixo (open-source) | Medio (modelo especifico) | Medio (API do provider) |
| **Maturidade** | N/A | Media (comunidade ativa) | Alta (papers publicados) | Alta (APIs estaveis) |

**Recomendacao: Hibrido — Vanna.ai como base + LLM API (Claude Sonnet) para geracao**

Justificativa: Vanna.ai oferece o framework de RAG para NL-to-SQL ja pronto, com suporte a training data por tenant. Combinado com Claude Sonnet para a geracao efetiva do SQL, obtemos alta precisao com baixo esforco de implementacao. O fallback para Gemini Flash em queries simples reduz custo.

| Decisao | Valor |
|---------|-------|
| Escolha | **Buy (Vanna.ai) + Build (orquestracao custom)** |
| Esforco estimado | 4-6 semanas |
| Risco | Baixo |

### 3.3 LLM Provider

| Criterio | Claude API (Anthropic) | Gemini API (Google) | Open-source Self-hosted (Llama 3, Mistral) |
|----------|----------------------|--------------------|--------------------------------------------|
| **Qualidade NL-to-SQL** | Excelente (melhor raciocinio logico) | Muito boa (Flash rapido, Pro preciso) | Boa (requer fine-tuning para equiparar) |
| **Custo por 1M tokens (input/output)** | $3.00 / $15.00 (Sonnet 4) | $0.10 / $0.40 (Flash) / $1.25 / $10.00 (Pro) | $0 (API) mas $2K-8K/mes em GPU |
| **Latencia (time to first token)** | ~500ms | ~200ms (Flash) / ~800ms (Pro) | ~300-1000ms (depende do hardware) |
| **Multi-idioma (PT-BR)** | Excelente | Excelente | Variavel (melhor em EN) |
| **Rate limits** | Generosos (Tier 2+) | Generosos (creditos GCP) | Ilimitado (self-hosted) |
| **Compliance / Privacidade** | SOC2, dados nao usados para treino | SOC2, GCP compliance | Total controle |
| **Custo de infra (GPU)** | $0 | $0 | $2,000-8,000/mes (A100/H100) |
| **Esforco de manutencao** | Zero | Zero | Alto (updates, scaling, monitoring) |
| **Vendor lock-in** | Medio | Medio (creditos GCP favorecem) | Nenhum |

**Recomendacao: Multi-provider — Gemini Flash (roteamento) + Claude Sonnet (SQL) + Gemini Pro (fallback)**

Justificativa: Usar Gemini Flash (mais barato e rapido) para classificacao de intent e insights simples. Claude Sonnet para a geracao de SQL (melhor raciocinio). Gemini Pro como fallback para queries complexas que precisam de contexto grande (1M token window). Self-hosted nao se justifica no horizonte de 3 anos dado o custo de GPU e manutencao vs APIs.

| Decisao | Valor |
|---------|-------|
| Escolha | **Buy (Multi-provider: Gemini + Claude)** |
| Esforco estimado | 2-3 semanas (abstraction layer) |

## 7. Recomendacoes Estrategicas

### Otimizacao de Custos — Quick Wins

| Acao | Impacto Estimado | Esforco |

| Decisao | Deadline | Impacto |
|---------|----------|---------|
| Definir mix de LLM providers (Claude vs Gemini) | Semana 2 do MVP | Impacta arquitetura da abstraction layer |
| Modelo de pricing (planos fixos vs pay-per-query) | Semana 4 do MVP | Impacta toda a monetizacao |
| Investimento em cache semantico | Mes 3 | Reduz 30-40% dos custos operacionais |
