# Bloco #3 — Valor e OKRs

> Definição de valor entregue, OKRs do MVP, modelo de negócio, pricing e métricas de sucesso para o Veezoozin.

---

## 1. Proposta de Valor Central

**"Pergunte para seus dados"** — O Veezoozin elimina a barreira técnica entre pessoas de negócio e seus próprios dados. Em segundos, qualquer colaborador obtém respostas visuais, contextualizadas e acionáveis, sem depender de times de dados ou conhecimento de SQL. [BRIEFING]

### Value Proposition Canvas (resumo)

| Dimensão | Descrição |
|----------|-----------|
| **Customer Job** | Obter respostas de negócio a partir de dados corporativos de forma autônoma e rápida [BRIEFING] |
| **Pain** | Dependência de time técnico (tickets de dias), respostas sem contexto, tabelas brutas sem insights [BRIEFING] |
| **Gain** | Resposta em segundos, gráficos automáticos, insights contextualizados pelo domínio do tenant [BRIEFING] |
| **Pain Reliever** | Conversão NL → SQL contextualizada, eliminação de filas e tickets, glossário de negócio por tenant [BRIEFING] |
| **Gain Creator** | Visualizações automáticas, sugestões de próximas perguntas, relatórios exportáveis, aprendizado contínuo [BRIEFING] |

### Impacto Mensurável Esperado

| Métrica de impacto | Antes | Depois (com Veezoozin) | Fonte |
|--------------------|-------|------------------------|-------|
| Tempo para obter resposta de dados | 2-5 dias (ticket) | < 5 segundos | [BRIEFING] |
| % de colaboradores que acessam dados | 5-10% (apenas técnicos) | 100% (qualquer colaborador) | [BRIEFING] |
| Qualidade do output | Tabela bruta CSV/Excel | Gráfico + insight + análise + sugestão | [BRIEFING] |
| Volume de tickets para time de dados | 100% (referência) | Redução de 60-80% | [BRIEFING] |

## 2. OKRs do MVP (12 semanas)

### OKR 1 — Produto funcional e validado

**Objetivo:** Entregar um MVP que demonstre o core do Veezoozin — consulta NL → SQL → visualização — com qualidade suficiente para os primeiros tenants.

| Key Result | Meta | Métrica | Fonte |
|-----------|------|---------|-------|
| KR1.1 | Precisão de query gerada ≥ 85% na primeira tentativa | % de queries SQL corretas sem correção manual | [BRIEFING] |
| KR1.2 | Latência de resposta < 5s para queries simples (até 3 JOINs) | P95 latency em segundos | [BRIEFING] |
| KR1.3 | Suporte funcional a 3 idiomas (PT-BR, EN-US, ES) | % de perguntas corretamente interpretadas por idioma ≥ 80% | [BRIEFING] |
| KR1.4 | Sugestão de prompts com taxa de aceitação ≥ 40% | % de sugestões clicadas / total de sugestões exibidas | [BRIEFING] |

### OKR 2 — Plataforma multi-tenant operacional

**Objetivo:** Suportar múltiplos tenants com isolamento, onboarding automatizado e controle de acesso.

| Key Result | Meta | Métrica | Fonte |
|-----------|------|---------|-------|
| KR2.1 | Suporte a pelo menos 5 tenants simultâneos com 50+ tabelas cada | Nº de tenants ativos sem degradação | [BRIEFING] |
| KR2.2 | Onboarding de novo tenant ≤ 2 horas (setup técnico + mapeamento de schema) | Tempo médio de onboarding | [INFERENCE] |
| KR2.3 | Zero incidentes de vazamento de dados entre tenants | Nº de incidentes de cross-tenant data leak | [BRIEFING] |
| KR2.4 | Controle de acesso a nível de registro/campo implementado para pelo menos 1 tenant piloto | Feature funcional e testada | [BRIEFING] |

### OKR 3 — Viabilidade econômica validada

**Objetivo:** Validar que o modelo de negócio é sustentável e que os custos de operação estão dentro do esperado.

| Key Result | Meta | Métrica | Fonte |
|-----------|------|---------|-------|
| KR3.1 | Custo de infraestrutura ≤ R$ 5K/mês para até 50 tenants | TCO mensal em GCP | [BRIEFING] |
| KR3.2 | Pelo menos 2 tenants pagantes (plano Starter ou superior) até o final do MVP | Nº de tenants com billing ativo | [INFERENCE] |
| KR3.3 | Custo médio por query ≤ R$ 0,05 (incluindo LLM API + compute) | Custo unitário por consulta | [INFERENCE] |
| KR3.4 | Unit economics positivo no plano Pro (margem bruta ≥ 60%) | Receita por tenant - custo variável por tenant | [INFERENCE] |

## 3. Modelo de Negócio

### 3.1 Tipo de Receita

**SaaS Subscription + Usage-based hybrid** — Assinatura mensal por plano (feature-gated) com componente de consumo (queries, storage, tenants adicionais). [BRIEFING][INFERENCE]

### 3.2 Planos e Pricing

| Plano | Free | Starter | Pro | Enterprise |
|-------|------|---------|-----|-----------|
| **Preço mensal** | R$ 0 | R$ 297/mês | R$ 897/mês | Sob consulta (a partir de R$ 2.997/mês) |
| **Tenants** | 1 | 1 | 3 | Ilimitado |
| **Queries/mês** | 50 | 500 | 3.000 | Ilimitado (fair use) |
| **Tabelas mapeadas** | 10 | 50 | 200 | Ilimitado |
| **Idiomas** | PT-BR apenas | PT-BR, EN-US | PT-BR, EN-US, ES | PT-BR, EN-US, ES + custom |
| **MCP / RAG externo** | — | — | 1 conexão | Ilimitado |
| **Controle de acesso por campo** | — | — | Básico (column-level) | Completo (row + column level) |
| **Exportação** | — | PDF | PDF, HTML | PDF, HTML, API |
| **Suporte** | Comunidade | E-mail (48h SLA) | E-mail + Chat (8h SLA) | Dedicado (4h SLA) + CSM |
| **Retenção de histórico** | 7 dias | 30 dias | 90 dias | 1 ano |
| **Glossário de negócio** | 20 termos | 100 termos | 500 termos | Ilimitado |
| **Usuários** | 1 | 5 | 20 | Ilimitado |
| **Fonte** | [BRIEFING][INFERENCE] | [INFERENCE] | [INFERENCE] | [INFERENCE] |

> [!note] Estratégia do plano Free
> O plano Free serve como **top-of-funnel** para aquisição. Recursos deliberadamente limitados (50 queries, 10 tabelas, apenas PT-BR) para demonstrar valor sem entregar a experiência completa. O objetivo é gerar conversão para Starter em até 14 dias. [BRIEFING][INFERENCE]

### 3.3 Add-ons (componente usage-based)

| Add-on | Preço | Disponível em |
|--------|-------|---------------|
| Pacote de 500 queries extras | R$ 97/mês | Starter, Pro |
| Tenant adicional | R$ 197/mês | Starter, Pro |
| Conexão MCP adicional | R$ 147/mês | Pro |

[INFERENCE]

### 3.4 Estratégia de Pricing

| Princípio | Aplicação |
|-----------|-----------|
| **Value-based pricing** | Precificação baseada no valor percebido (horas economizadas, tickets eliminados), não no custo de infraestrutura |
| **Land and expand** | Free → Starter rápido; Starter → Pro quando o tenant cresce em tabelas e usuários; Pro → Enterprise quando precisa de compliance e integrações |
| **Transparent metering** | Dashboard de consumo em tempo real para o admin do tenant, sem surpresas no billing |
| **Annual discount** | 20% de desconto para contratos anuais (incentivo a comprometimento e redução de churn) |

[INFERENCE]

## 6. Riscos de Valor

| Risco | Impacto | Probabilidade | Mitigação | Fonte |
|-------|---------|---------------|-----------|-------|
| Precisão de query abaixo de 85% — usuários perdem confiança | Alto | Média | Feedback loop com thumbs up/down; fallback para sugestão de query alternativa; refinamento contínuo de prompts | [BRIEFING][INFERENCE] |
| Custo de LLM API cresce com adoção e inviabiliza o Free tier | Alto | Média | Cache agressivo de queries similares; query templates; limites rígidos no Free; monitoramento de custo por query | [INFERENCE] |
| Concorrentes enterprise (ThoughtSpot, Tableau) adicionam NL features | Médio | Alta | Diferenciação via multi-idioma nativo, contexto por tenant, preço acessível para PMEs, MCP integration | [BRIEFING][INFERENCE] |
| Churn alto no Starter por features insuficientes | Médio | Média | Iteração rápida no produto; entrevistas de churn; ajuste de limites entre tiers | [INFERENCE] |
| Clientes enterprise não confiam em SaaS para dados sensíveis | Alto | Média | LGPD compliance desde o dia 1; SOC2 roadmap; opção de dedicated instance no Enterprise | [BRIEFING][INFERENCE] |

## 7. Resumo Executivo de Valor

O Veezoozin resolve um problema real e mensurável: **a democratização do acesso a dados corporativos**. O modelo SaaS tiered (Free → Starter → Pro → Enterprise) permite captura de mercado via freemium com expansão orgânica. [BRIEFING][INFERENCE]

**Números-chave:**

- **Investimento MVP:** ~R$ 196K em 12 semanas [INFERENCE]
- **Break-even operacional:** Mês 3 pós-launch [INFERENCE]
- **Payback total:** Mês 8-10 pós-launch [INFERENCE]
- **ARR projetado (M12):** ~R$ 460K [INFERENCE]
- **LTV/CAC alvo:** ≥ 8:1 [INFERENCE]

O modelo é viável desde que a precisão de queries (≥ 85%) e os custos de LLM (≤ R$ 0,05/query) sejam mantidos dentro dos targets. [BRIEFING][INFERENCE]
