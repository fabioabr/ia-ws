# File Naming

Standard naming convention for files, folders, skills, and agents.

## Standard

- **Format:** kebab-case, always in English
- **Characters:** lowercase letters, numbers, hyphens only
- **No** spaces, underscores, camelCase, or accented characters

## Examples

| Type | Example |
|--------|-------------------------------|
| File | `document-management.md` |
| Folder | `project-boundaries/` |
| Skill | `/validate-doc` |
| Agent | `code-review` |
